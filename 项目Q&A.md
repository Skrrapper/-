- ## **你做的项目是什么？名称、简介、愿景、功能、使用人群**

  名称：**青言速递**；

  简介：是一款面向校园场景的智慧通知平台，为解决校园信息传递中的“**碎片化、冗余化、滞后化**“问题，构建统一、权威、高效的**信息发送与接收平台**。

  愿景：打造校园**唯一且权威**的信息发布平台，实现“**发布-审核-推送-反馈-归档**”的全过程，使得校园信息**标准而又高效**。

  功能：

  **（1）用户账号管理**

  **（2）公告的创建、编辑与分类**

  **（3）公告的搜索与显示**

  **（4）后台管理与数据统计**

  使用人群：

  （1）**学生**：主要使用者，查看公告与评论

  （2）**教师/发布者**：公告发布者，编写与发布

  （3）**管理员**：负责审核、管理用户与统计数据

- ## **项目用了什么技术？架构？**

  **前端**：Vue3 

  **后端**：Spring Boot 

  **通信**：RESTful API 

  **数据库**：MySQL，关键字段建立索引

  **身份认证**：JWT Token

  

  **总体架构：**

  使用B/S模式 + 前后端分离架构，前后端通过RESTful API 进行通信。确保系统结构清晰，高内聚低耦合



- ## **详细设计？**

  **1.用户认证与权限控制**

  使用 JWT（JSON Web Token） 实现无状态登录； 前端请求统一携带 Token，后端通过 Spring Security 校验； 根据用户角色动态授权访问接口。 

  

  **2.公告发布与审核流程**  

  教师创建公告后，若分类需审核 → 状态置为  管理员审核后可将状态改为  pending ； published 或  rejected ； 公告发布后自动写入 Redis 缓存，并推送给订阅用户。  

  

  **3.评论与统计设计**  

  评论支持多级嵌套（通过 parent_id 实现）； 浏览量统计使用 Redis 自增计数，并定期异步同步到 MySQL； 数据统计通过 SQL 聚合 + Redis 缓存结合实现，使用 ECharts 展示结果。 

  

  **4.推送机制**  

  通过 WebSocket 实现实时消息推送； 邮件用于系统级通知（如审核结果、重要公告）。
